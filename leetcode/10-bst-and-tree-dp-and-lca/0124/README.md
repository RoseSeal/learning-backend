# 124. Binary Tree Maximum Path Sum
耗时 10 分钟。

## 我的思考
一开始想到遍历节点，返回两个值：
- 通过本节点的最大值
- 不通过本节点的最大值

然后答案取根节点这两个值的较大者。

但很快发现这样不好更新。因为“通过本节点的最大值”其实又分成两种情况：
- 路径终止于本节点，后续还可以继续向父节点累加
- 路径真正穿过本节点，左右两侧都接入，这种情况不能再继续向父节点累加

于是修改为：返回“可以继续累加的最大值”和“不可继续累加的最大值”。

再进一步想到，其实“不可继续累加的最大值”没有必要作为返回值传给父节点，因为它对父节点的计算没有价值。

最终思路是：遍历每个节点时，只返回**可向上累加的最大值**；同时用一个全局变量更新答案。

对于当前节点：
- 左子树返回左侧可向上累加的最大值
- 右子树返回右侧可向上累加的最大值
- 当前节点向父节点返回的值只能选择一侧，因此返回  
  `max(curr + 左可累加值, curr + 右可累加值)`

而当前节点作为路径最高点时，可以同时接入左右两侧，所以用：

`curr + left + right`

来更新全局答案。

另外，左右子树如果贡献为负数，就不如不选，所以先和 `0` 取最大值。

本题是 3 个月前刷过的题，属于隐约有印象；当时也是这个解法 AC 了，但耗时比现在更久一些。

## 卡点
1. 一开始把“通过本节点的最大值”当成单一概念处理，但它实际上包含“可继续向上延伸”和“不能继续向上延伸”两种不同语义。

## 关键点
1. 递归返回值表示：**从当前节点出发，向上能够提供的最大单边路径和**。
2. 全局答案表示：**任意节点作为路径最高点时的最大路径和**。
3. 当前节点更新答案时，可以同时使用左右子树：
   `val + left + right`
4. 当前节点返回给父节点时，只能选一边：
   `max(val + left, val + right)`
5. 子树贡献为负时直接舍弃，因此写成：
   `left = max(0, search(curr.left))`
   `right = max(0, search(curr.right))`

## 复杂度
- 时间：`O(n)`
- 空间：`O(h)`，其中 `h` 为树高；递归栈最坏情况下为 `O(n)`

## 注意
1. 返回值和全局答案的语义不同，一个是“可向上延伸”，一个是“当前点作为拐点的完整路径”。
2. `return` 时不能返回 `val + left + right`，因为这条路径已经同时用了左右两边，不能再向父节点继续延伸。

## 总结
1. 树递归题先区分：什么信息要返回给父节点，什么信息只在当前节点用于更新答案。
2. 若路径要继续向父节点延伸，返回值必须保持为单链，不能分叉。
3. 若某个局部结构能形成完整答案，但不能继续向上传递，就在当前节点更新全局答案。
4. 负贡献如果可以被主动舍弃，就直接裁成 0。
