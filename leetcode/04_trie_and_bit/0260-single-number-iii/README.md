# 260. Single Number III

## 我的思考
首先肯定是先异或一遍，我们得到的是两个答案数字的异或结果。现在问题变成：**如何把这两个数字区分开**。

首先，两个数字异或结果本身的信息，其实**不足以直接还原出这两个数**，我们还需要额外信息。并且题目限制必须做到 `O(n)` / `O(1)` 的时空复杂度，这意味着我们最多只能再常数次遍历数组。

AI 提示：异或结果中，哪一位是 `1`？

于是继续思考：遍历 `num`，这个 `1` 是否意味着这个 `num` 有没有成为候选答案的潜力？  
所以假设两个答案的异或结果是 `xor`，那么 `xor & num` 会有性质吗？

但是这里发现不太行。  
`xor` 里的 `1` 可能来自 `ans1`，也可能来自 `ans2`，并且 `ans1` 和 `ans2` 里的 `1` 也不一定会完整保留在 `xor` 里。看起来这条路用不了。

AI 提示：`xor` 中某一位是 `1`，说明 `ans1` 和 `ans2` 在这一位上不同。

于是想到这个性质：`xor` 里的所有 `1`，都代表 `ans1` 和 `ans2` 在这一位上不同。  
那该如何利用这个性质？

最简单的想法是：给每一位打上分组记号，然后所有分组记号都不同的两个数字就是答案。  
但这样有两个问题：

1. 空间复杂度超了。
2. 这些 `1` 真的足够区分 `ans1` 和 `ans2` 吗？

例如：
- `ans1 = 100101`
- `ans2 = 100010`
- 那么 `xor = 000111`

但如果数组里还有两个数：
- `110101`
- `110010`

它们在低三位上也同样都不同。  
所以如果试图利用“所有不同位”去做更复杂的标记，并不能直接锁定答案。

如果只是简单地分成两组，那么如何从这两组里提取出各自的 `ans` 又是一个问题。

我知道了：
**相同的值必然会被分在同一组。**

所以，只要找到 `xor` 中任意一个为 `1` 的位，把所有数字按这一位是 `0` 还是 `1` 分成两组，那么：
- 两个答案一定会被分到不同组
- 所有成对出现的相同数字一定会被分到同一组

这样一来，假设分为 `a`、`b` 两组，只需要分别对组内做异或，得到的结果就是 `ans1` 和 `ans2`。

## 卡点
1. 一开始虽然知道先整体异或，但不知道如何从 `xor` 结果继续推进。
2. 误以为需要利用 `xor` 中所有为 `1` 的位做更复杂的区分，结果发现既不满足 `O(1)` 空间，也不能唯一锁定答案。
3. 没有立刻意识到“只需要任选一个为 `1` 的位做二分”，并且“相同数字一定进同一组”这个性质正好可以保证组内异或有效。

## 关键点
1. 先整体异或，得到 `xor = ans1 ^ ans2`。
2. `xor` 中任意一个为 `1` 的位，都说明 `ans1` 和 `ans2` 在这一位上不同。
3. 用这一位作为分组依据，可以把所有数分成两组。
4. 相同数字二进制完全相同，所以一定会被分到同一组，组内异或后自动抵消。
5. 两个只出现一次的答案会进入不同组，因此两组各自异或的结果就是答案。

## 复杂度
- 时间：`O(n)`
- 空间：`O(1)`

## 注意
1. 技巧 `divider = oxr & -oxr` 的作用是取出 `oxr` 二进制下最低位的那个 `1`。
2. 最后返回的两个答案顺序不固定，因为题目通常不要求顺序。
