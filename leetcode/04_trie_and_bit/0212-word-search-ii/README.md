# 212. Word Search II

## 我的思考
显然，可以先将 `words` 转化为 Trie 结构，然后遍历二维数组，对每一个字符去 Trie 中搜索，并对上下左右四个方向做深度优先搜索。一旦搜到单词，就将 `word` 保存下来。

这看起来是可行的，而且也基本是最优思路，没有看到明显不必要的开销。

进一步优化：将已经搜到的 `word` 从 Trie 里删去。  
这应该是一个值得做的优化。

一开始思考如何实现 `delete`：  
也许需要记录 Trie 中每一个节点实际上有多少个单词“通过”，每找到一个单词就将这个数量减一，数量为 `0` 的节点直接置为 `null`。

AI 提示：这太重了，不需要。

那如果不记录数量，是否可以直接剪掉最靠近叶子部分、且没有分支的节点？  
接着思考：那该如何找到这个节点？

也许要依靠回退。  
甚至想过，要不要把数组实现的边改成 `HashMap` 实现，这样就不用遍历来判断是否有分支，而是可以直接通过 `size()` 判断。

继续想下去，会发现：最靠近叶子部分的那个分支节点，在递归回退时一定会经过。问题在于，如何识别它。

同时也发现，其实删除功能必须加，否则 `ans` 会出现重复元素。

于是继续思考如何 `delete`：  
也许可以把 DFS 的返回值设为 `boolean`。找到一个 `isEnd` 后返回 `true`，然后每一层判断，如果有一个方向返回了 `true`，就进一步判断本层是否可以删除，也就是判断是否有分叉；如果不能删除，就继续返回 `true`，否则就删除并返回 `false`。

AI 提示：思考一下什么时候可以删除本节点。

然后想到了。  
这里的不变性是：

**当本节点无子节点、且不是单词结尾时，就可以删除。**

所以只需要每次找到一个答案时，顺手把 `isEnd = false`，就可以在递归回退过程中优雅地删除那些已经无用的节点。

另外，第一版代码还出现过两个问题：

1. 遇到 `isEnd` 时，没有继续搜索四个方向。
2. 没有标记访问过的格子。

这两个问题本质上都是粗心导致的。

AI 提示的技巧：

1. 用 `board` 数组本身做标记，省掉 `visited` 数组的开销。
2. 在 Trie 中直接存储 `word`，省掉 `StringBuilder` 的开销。

## 卡点
1. 一开始虽然确定了 Trie + DFS 的主框架，但对“找到答案后如何删除 Trie 中无用节点”没有想清楚。
2. 曾经把删除逻辑想复杂了，试图给每个 Trie 节点维护“经过单词数”之类的信息，实际上没有必要。
3. 一度考虑通过 DFS 返回值来层层上传删除信号，思路能做，但不够优雅。
4. 第一版实现还漏掉了两个关键细节：  
   - 找到单词后仍然要继续向四个方向搜索。  
   - 必须标记当前格子已访问，防止重复使用。

## 关键点
1. 用 Trie 压缩 `words`，避免对每个单词单独搜索。
2. 从 `board` 的每个格子出发做 DFS，在 Trie 上同步推进。
3. 当当前字符在 Trie 中没有后继时，立即剪枝返回。
4. 用 `board[i][j] = '#'` 这种原地修改方式标记访问，省掉 `visited` 数组。
5. 在 Trie 节点中直接存 `word`，当走到单词结尾时可以直接收集答案，不需要额外维护路径字符串。
6. 为避免重复加入答案，找到后直接将 `next.word = null`。
7. 回退时，如果某个子节点满足“无子节点且不是单词结尾”，就可以从父节点中删除，进一步剪枝。

## 复杂度
- 时间：`O(mn * 4 * 3^(L-1))`，其中 `m、n` 是棋盘大小，`L` 是单词最大长度。构建 Trie 还需要 `O(所有单词总长度)`。
- 空间：`O(所有单词总长度 + L)`，其中前者是 Trie 的空间，后者是 DFS 递归栈深度。

## 注意
1. 找到一个单词后不能立刻停止当前路径搜索，因为更长的单词可能以它为前缀。
2. 必须标记当前格子已经访问，否则同一路径中可能重复使用同一个格子。
3. 用 `next.word = null` 去重非常关键，否则相同单词可能被重复加入答案。
4. 回退阶段的删除条件是：**该节点没有子节点，并且不是单词结尾**。
5. 这题里使用 `HashMap<Character, Trie>` 实现 `next`，可以方便通过 `size()` 和 `remove()` 做剪枝；如果字符集固定，也可以用数组实现，但删除和判空处理会稍微不同。
