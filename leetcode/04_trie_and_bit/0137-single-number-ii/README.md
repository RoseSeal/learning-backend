# 137. Single Number II

## 我的思考
一开始看到这道题，首先想到的还是异或。

但异或只能做到**两两抵消**，而这题是**出现三次**。  
于是先尝试：

思路1：重复做一次异或。  
不行，因为如果数字出现3次，会变成6次；但只出现1次也会变成2次，最终所有内容都会被消去。

继续思考：  
异或的本质是**不进位的二进制加法**，它实际上是在统计**每一位的奇偶性**。

那么能不能构造一个**模三的二进制加法**？

也就是状态循环：

```
00 → 01 → 10 → 00
```

表示某一位出现次数对3取模的状态变化。

问题来了：  
难道要对每一个 bit 位单独做模三加法吗？

这样时间复杂度是：

```
32 * n
```

虽然仍然是 O(n)，但感觉太低效。  
就像我们做模二运算时，并不会真的循环32次逐位处理。

**一定存在某个设计好的表达式，可以直接得到最终结果。**

---

（AI 提示）
> 用两个数 `ones` 和 `twos`。

于是想到：

```
ones:记录每一个出现一次的bit：
    ones中的每一个bit，若为1，表示这个bit出现次数是1+3n次； 若为0，表示这个bit出现次数是3n次或者2+3n次；
twos:记录每一个出现二次的bit：
    twos中每一个bit，若为1，表示这个bit出现次数是2+3n次 若为0，表示这个bit出现次数是3n次或者1+3n次； 
不考虑初始状态怎么实现，就类似dp思想，只要弄清楚怎么更新，初始实现可以倒推（比如都等于0）；
```

---

接下来整理状态流转表。

现在新出现一个 `num`：

（只需要考虑 num = 1，因为 num = 0 不影响状态）

```
num = 1 时：

ones: 0 → 1 → 0 → 0
twos: 0 → 0 → 1 → 0
```

---

接下来需要把这个**状态流转压缩成表达式**。

这一步卡了很久，因为我对 `& ~ |` 这些位运算的理解还不够深入。

---

（AI 提示）
> 使用 `&` 和 `~`。

我开始对照状态变化表推表达式。

最终写出：

```
ones = (ones ^ num) & ~twos
```

理解为：

- `ones ^ num`：先按照异或更新
- `& ~twos`：`twos` 中的1会禁止ones的本bit位更新

写出 `ones` 的更新之后，  
`twos` 的更新逻辑实际上是完全对称的：

```
twos = (twos ^ num) & ~ones
```

---

最后：

```
return ones
```

因为 `ones` 中记录的正是 **出现次数 mod 3 为 1 的位**。
