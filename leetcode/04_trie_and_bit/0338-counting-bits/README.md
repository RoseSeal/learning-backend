# 338. Counting Bits

## 我的思考
先观察序列：

`000`  
`001`  
`010`  
`011`  
`100`  
`101`  
`110`  
`111`

可以发现：
- 最低位的 `1` 是隔 `1` 位加 `1` 位；
- 更高一位的 `1` 是隔 `2` 位加 `2` 位；
- 再高一位是隔 `4` 位加 `4` 位。

这也符合二进制的进制原理。  
如果把这一段规律直接描述成代码，就是按位刷入，大概类似这样的伪代码：

`interval = 1`  
`while interval < n:`  
`    for i = interval; i < n; i += interval:`  
`        for j = i; j < i + interval; j++:`  
`            ans[j]++`  
`    interval *= 2`

这样做本质上是按位贡献去刷，时间复杂度是 `O(n log n)`，复杂度偏高。

继续考虑，可以发现：`i >> 1` 之后的 `1` 的个数，其实就等于 `ans[i >> 1]`，而当前 `i` 相比 `i >> 1` 多出来的，只是最低位这一位。  
所以可以直接得到：

`ans[i] = ans[i >> 1] + (i & 1)`

## 卡点
一开始想到的是按二进制每一位的贡献去整体刷入，但这样复杂度偏高。

## 关键点
1. `i >> 1` 相当于去掉 `i` 的最低位，并且等于`ans[i >> 1]`

## 复杂度
- 时间：`O(n)`
- 空间：`O(n)`
