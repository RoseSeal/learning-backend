# 16. 3Sum Closest

## 我的思考
这题思路比较直白。

先排序，然后固定第一个数，剩余部分用双指针寻找最接近 `target` 的三数之和，整体是 `O(n^2)`。

实现时没有直接维护“当前最接近的三数和”，而是让 `ans` 维护 `target - 当前三数和`。  
这样在比较时，只需要判断这个差值的绝对值是否更小；最后返回 `target - ans` 即可。

同时结合排序结果做了两类剪枝：
- 如果当前最小数搭配最大的两个数仍然小于 `target`，那么这一轮能取得的最优结果就是这三个数之和
- 如果当前最小数搭配后面最小的两个数已经大于 `target`，那么这一轮能取得的最优结果就是这三个数之和，并且后面也不可能更优，可以直接结束

## 关键点
1. 排序后，固定一个数，再用双指针做 `Two Sum` 式逼近。
2. `ans` 维护的是 `target - 三数和`，这样更新和比较都比较顺手。
3. 利用排序结果，可以先判断当前轮的最大和、最小和，提前剪枝。

## 复杂度
- 时间：`O(n^2)`
- 空间：`O(log n)` 或 `O(1)`，取决于排序实现

## 注意
1. 这题维护“差值”通常比直接维护“三数和”更易读，也更好写。
2. 命中 `curr == 0` 时可以直接返回 `target`。
