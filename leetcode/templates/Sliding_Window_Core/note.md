# 滑动窗口核心总结

## 模板选择

### 模板一：固定长度窗口
- 窗口大小天然固定，一般是 `k` 或 `pattern.length()`
- `right` 扩张一个，`left` 同步跟进一个
- 适用：定长子数组最值、排列 / 异位词匹配、固定长度统计
- 代表题：`643`、`1456`、`567`

### 模板二：变长窗口求最值
- `right` 持续扩张
- 一旦违反约束，`left` 收缩到重新满足条件
- 适用：最长 / 最大合法子数组、最多修改 `k` 次、预算限制
- 代表题：`1004`、`424`、`1208`

### 模板三：变长窗口做计数
- 核心不是“当前窗口是否合法”，而是“当前 `right` 结尾新增了多少答案”
- 当窗口合法后，要思考本轮能否批量统计答案
- 代表题：`713`、`992`

---

## 固定窗口和变长窗口怎么区分
- 如果题目要求的子串 / 子数组长度天然不变，优先考虑固定窗口
- 如果题目约束是“窗口内最多 / 至多 / 不超过”，通常是变长窗口
- 如果题目问的是子数组个数，不要只想维护合法窗口，还要想怎么收集答案

---

## 收集答案的方式

### 最值类
- 窗口恢复合法后，用 `right - left + 1` 更新答案
- 典型题：`1004`、`424`、`1208`

### 计数类
- 当窗口合法时，思考“以 `right` 结尾的合法子数组有多少个”
- 若窗口内任意起点都合法，则：
```java
ans += right - left + 1;
```
- 典型题：`713`

### 恰好 K 类
- 两种常见思路：
  - 直接维护“恰好 K”并统计冗余前缀
  - 转化成：`exactly(k) = atMost(k) - atMost(k - 1)`
- 第二种通常更稳、更通用
- 典型题：`992`

---

## 不变量设计
滑动窗口的本质是维护一个**区间不变量**。

- `1004`：窗口内 `0` 的个数 `<= k`
- `1208`：窗口内总代价 `<= maxCost`
- `424`：窗口长度 `- maxCount <= k`
- `992 atMostK`：窗口内不同整数个数 `<= k`

设计窗口时先想清楚：
- 窗口里维护的量是什么？
- 什么条件下窗口合法？
- `right` 扩张后，`left` 应该收缩到什么程度？
- 答案是在“窗口合法时更新”，还是“统计本轮新增答案”？

---

## 关键理解(弱不变量)

### 1. 辅助量不一定要精确回退
- `424` 中的 `maxCount` 不需要随着 `left` 收缩而回退
- 它可以表示历史最大值，而不是当前窗口真实最大值
- 只要不会漏掉最优答案，就允许这种“松维护”

### 2. 严格合法不是唯一目标
- 有些题窗口可以维护成“仍有成为答案的潜力”
- `1208` 就可以利用这个性质，省掉显式 `ans`

### 3. 计数题优先看每个 right 的贡献
- 不要只盯着一个合法窗口
- 要问：当前 `right` 固定时，有多少个左端点能组成答案
- `713` 和 `992` 都是这个思路

### 4. 弱不变量 / 松维护的使用条件
- 辅助量不一定要精确反映当前窗口状态
- 关键在于：即使它不精确，窗口判定仍然不会漏掉最优答案，或不会影响最终答案正确性
- 典型例子：`424` 中的 `maxCount`

---

## 常见坑
1. 没先判断窗口长度是否固定，把定长题写成了变长题
2. 只会维护合法窗口，但不会统计“同一个 `right` 对应多少答案”
3. `exactly k` 没想到转成 `atMost(k) - atMost(k - 1)`
4. 对辅助变量语义不清楚，比如 `unsatisfied`、`diffNums`、`redundantPre`
5. 默认要求每个辅助量都精确反映当前窗口状态，导致思路变重
6. 收缩条件写错，应该是“违反约束就收缩”，不是“感觉差不多就收缩”

---

## 题型识别与本质
- 滑动窗口的本质不是”双指针一起移动”，而是维护一个连续区间的不变量，并在移动过程中高效更新答案
- 子串 / 子数组问题，且要求连续 → 滑动窗口
- 最长、最大、最短的连续区间 → 先想变长窗口
- 长度固定的连续区间统计 → 先想固定窗口
- 子数组个数统计 → 先想”每个 `right` 的贡献”
- 恰好 `k` 个不同元素 / 恰好满足某条件 → 先想能否转成 `atMost`

---

## 面试解释算法思路
- 我维护的是一个连续窗口 `[left, right]`。
- `right` 负责扩张窗口，把新元素纳入统计。
- 如果窗口违反约束，我就移动 `left` 收缩窗口，直到恢复不变量。
- 如果题目求最值，我在窗口合法时更新答案。
- 如果题目求个数，我统计当前 `right` 结尾时新增了多少合法子数组。

## 模板

- 固定长度窗口
```java
for (int right = 0; right < n; right++) {
    add(nums[right]);
    if (right >= k) {
        remove(nums[right - k]);
    }
    if (right >= k - 1) {
        updateAnswer();
    }
}
```

- 变长窗口求最值
```java
int left = 0;
for (int right = 0; right < n; right++) {
    add(nums[right]);
    while (!valid()) {
        remove(nums[left]);
        left++;
    }
    updateAnswer(right - left + 1);
}
```

- 变长窗口做计数（at most）
```java
int left = 0;
int ans = 0;
for (int right = 0; right < n; right++) {
    add(nums[right]);
    while (!valid()) {
        remove(nums[left]);
        left++;
    }
    ans += right - left + 1;
}
```

- 恰好 K
```java
int exactlyK(int[] nums, int k) {
    return atMost(nums, k) - atMost(nums, k - 1);
}
```
