# 25. Reverse Nodes in k-Group

## 我的思考
思路比较直白，就是按组处理链表，每次判断当前是否还存在一组长度为 `k` 的区间，如果存在，就对这一组做反转；如果不存在，就直接返回结果。

这题上次刷已经是半年前，解法基本上完全不记得了，只记得之前做过，因此这次基本可以视为接近初见。

## 关键点
1. 分组反转时，需要明确区间边界，反转的是 `[begin.next, end)` 这一段。
2. 反转链表本质上还是经典的三指针操作，`curr` 是当前处理指针。

## 复杂度
- 时间：`O(n)`
- 空间：`O(1)`

## 注意
1. 命名还不够直观，主函数和辅助函数中的 `begin` 实际不是同一个语义，容易增加理解成本。
2. 反转操作需要三指针配合，`curr` 表示当前操作指针。
3. 推荐命名：
 - 主流程里：`groupBegin`,`groupPre`,`groupEnd`,`nextGroupBegin`。
 - reverse里：`pre`,`curr`,`next`。
