# 909. Snakes and Ladders

## 我的思考

第一反应是 **BFS 最短路**：

- 每个编号 `1 ~ n^2` 本质上都是图中的节点
- 从当前节点 `curr`，可以走到 `curr + 1 ~ curr + 6`
- 如果落点有蛇/梯，本质都是 **强制跳转（redirect）**
- 每次掷骰子的代价固定为 `1`

因此本质是：

**implicit graph + unweighted shortest path → BFS**

一开始试图定义“有价值的节点”：

- 所有可以跳转的节点显然有价值
- 对于不能跳转的节点，尝试只保留最远节点或部分节点

但很快发现这种启发式剪枝不可靠：

- 当前看起来普通的节点，后续可能连接到更优的梯子
- `curr + 6` 也可能本身存在强制跳转，直接特殊处理容易出错

最终放弃人为筛选：

- 直接枚举 `curr + 1 ~ curr + 6`
- 所有合法节点都尝试入队
- 使用 visited 去重

这样逻辑最稳定。

另外需要将编号转换为棋盘坐标：

- 编号从左下角开始
- 每一行方向交替变化

这部分需要单独写 `num2index()` 映射函数。

## 卡点

### 1. 错误尝试“筛选有价值节点”

一开始试图提前判断哪些 next 值得入队：

- 只保留跳转节点
- 或者只保留最远普通节点

本质是过早做启发式剪枝。

问题：

无法保证局部最优等于全局最优。

正确做法：

**完整枚举合法邻居 + visited 去重**

---

### 2. 返回答案时机错误

一开始写成：

```java
if (curr + 6 >= end)
```

这样遗漏：

* 落点本身不是终点
* 但落点通过蛇/梯跳转到终点

---

### 3. `num2index()` 推导耗时较长

* 行号从底部开始计算
* 每行方向交替变化

需要先手动举例验证，再写公式。

---

## 关键点

### 1. 本质是隐式图最短路

```text
编号节点 + 固定边 + redirect + BFS
```

---

### 2. 有 redirect 的题，终点判断放在 redirect 后

错误：

```text
next == end
```

正确：

```text
dest == end
```

---

### 3. 不要过早做启发式剪枝

BFS 中：

* 完整枚举合法邻居
* visited 去重

通常比手动筛选更稳。

---

### 4. 编号映射问题先举例，再写公式

例如：

```text
1 -> (n-1,0)
2 -> (n-1,1)
...
n -> (n-1,n-1)
n+1 -> (n-2,n-1)
```

先验证规律，再公式化。

---

## 复杂度

* 时间：`O(n²)`

  * 每个编号最多访问一次
  * 每次最多扩展 6 条边

* 空间：`O(n²)`

  * queue
  * visited

* 避免无意义启发式剪枝
