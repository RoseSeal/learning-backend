# 934. 最短的桥

## 我的思考
本题可以先找到第一片陆地，再从第一片陆地向外进行 BFS 扩散，寻找抵达第二片陆地的最短距离。

实现思路是：先找到第一片陆地的边缘节点，再将这些边缘节点作为 BFS 初始队列向外扩散。该思路本身可行，但实现时完全按照脑中的过程一比一复刻，导致状态设计较复杂，需要区分本岛、边缘、已访问海水等多种状态。

后续意识到，其实有更简单的实现方式：

1. 可以两遍 BFS：
   - 第一遍标记第一座岛；
   - 第二遍在标记后的第一座岛中寻找边缘节点，再从边缘节点开始扩散。

2. 也可以直接将第一座岛的全部节点作为 BFS 初始队列：
   - 第一遍搜索时标记整座第一岛，并将所有第一岛节点加入队列；
   - 随后直接从整座第一岛向外做多源 BFS；
   - 内部节点由于周围仍是已访问岛屿，不会真正向外扩散，实际起作用的仍然是边缘节点。

这两种方式与原本写法在时间复杂度和空间复杂度上相同，但实现复杂度和阅读门槛显著降低。

## 卡点
- 原始写法过度贴合脑中的“先找边缘，再扩散”的过程，导致代码状态较多。
- 使用原地修改时引入了多个状态值，例如 `-1`、`-2`，使数组语义变得复杂。
- 为了显式找边缘，增加了实现复杂度，但并没有带来复杂度级别上的收益。
- 需要注意：BFS 队列本身最坏仍可能达到 `O(n^2)`，因此省去 visited 数组主要是常数优化，不改变渐进空间复杂度。

## 关键点
- 本题核心是“两阶段图搜索”：
  1. 找到并标记第一座岛；
  2. 从第一座岛出发做多源 BFS，第一次遇到第二座岛时的层数就是答案。

- 不一定需要显式寻找边缘节点。  
  可以直接把第一座岛全部加入 BFS 初始队列，边缘会在 BFS 扩散过程中自然生效。

- 原地修改可以使用，但状态语义应尽量简单。  
  推荐只使用少量清晰状态：
  - `0`：未访问海水；
  - `1`：未访问陆地；
  - `2`：第一座岛或已经扩散访问过的位置。

- 若某个优化只减少常数，却明显增加状态复杂度和阅读门槛，面试中通常不值得。

## 复杂度
- 时间：`O(n^2)`  
  每个格子最多被访问或入队一次。

- 空间：`O(n^2)`  
  最坏情况下 BFS 队列可能存储 `O(n^2)` 个格子。若使用递归 DFS 标记第一座岛，递归栈最坏也可能达到 `O(n^2)`。

## 注意
- 从第一座岛整体出发做 BFS，比单独寻找边缘节点更简洁。
- 第一次在 BFS 扩散中遇到值为 `1` 的格子，即表示到达第二座岛，可以直接返回当前距离。
- 原地修改虽然可以减少额外 visited 数组，但不要让原数组承担过多语义，否则容易增加 debug 难度。
- 写代码前应先判断：当前脑中思路是否需要完整复刻，还是可以用更简单的程序结构表达同样逻辑。
