# 1004. Max Consecutive Ones III

## 我的思考
维护**变长滑动窗口**，窗口内最多有k个0，记录窗口最大值。 
right一直前进，
left前进的条件是：right新进入了一个0，left需要前进直到退出一个0。 

**不变量是：窗口内只能有k个0**

在代码实现里，使用 `zeroCount` 记录当前窗口内 `0` 的个数。  
每次 `right` 扩张时，通过 `nums[right] ^ 1` 技巧来直接判断当前数是不是 `0`

当 `zeroCount > k` 时，说明当前窗口不合法，此时不断移动 `left`，并同步减少窗口左侧移出元素对 `zeroCount` 的贡献，直到窗口重新合法。

每次窗口合法时，用 `right - left + 1` 更新答案即可。

## 卡点
无

## 关键点
1. 这是典型的**变长滑动窗口**问题。
2. 不变量是：**窗口内 `0` 的个数不能超过 `k`**。
3. `right` 负责扩张窗口，`left` 负责在窗口失效时收缩窗口。
4. 窗口一旦重新合法，就可以用当前窗口长度更新答案。
5. 虽然代码中有 `for` 嵌套 `while`，但时间复杂度仍然是 `O(n)`，因为每个元素最多只会被 `left` 和 `right` 各访问一次。

## 复杂度
- 时间：`O(n)`
- 空间：`O(1)`

## 注意
1. `nums[right] ^ 1` 这种写法可以用，但更适合表述为个人习惯，而不是强调成“为了避免 if 导致的指令流水线预测损失”。这种性能收益很低，也很难在这里验证，属于典型的过度优化叙事。如果追求最佳可读性，直接写成判断当前元素是否为 `0` 会更自然。
