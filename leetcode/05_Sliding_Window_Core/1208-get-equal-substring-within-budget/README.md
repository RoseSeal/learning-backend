# 1208. Get Equal Substrings Within Budget

## 我的思考
这题是标准滑动窗口。

窗口内维护当前总代价 `cost`，其中每一位的代价是 `abs(s[i] - t[i])`。  
`right` 右移时加入新代价；如果 `cost > maxCost`，就左移 `left`，移除左端代价，使窗口重新合法。

这是最直接的写法，语义是：**窗口始终严格合法**。

进一步可以发现，这题不一定要显式维护 `ans`，也不一定要用 `while` 把窗口完全缩回合法状态。

因为每轮只新增一个字符代价，窗口长度最多只增加 `1`。  
所以如果超预算，只需要让 `left` 同步右移一步即可，不必持续收缩。

这样不变量的语义就从“窗口严格合法”，变成了“窗口仍然具有候选潜力的合法”。

因此第二版可以写成更简洁的形式：
- `right` 每轮右移
- 超预算时，`left` 右移一步
- 最终答案直接是 `t.length() - left`

## 关键点
1. 每个位置的代价是 `abs(s[i] - t[i])`。
2. 窗口维护区间总代价 `cost`。
3. 第一版是严格合法窗口。
4. 第二版利用了“每轮窗口最多只增 1”的性质，省掉了 `ans` 和 `while`。

## 复杂度
- 时间：`O(n)`
- 空间：`O(1)`

## 注意
1. 第二版更简洁，但前提是对窗口不变量理解清楚。
2. 第一版更通用，第二版更体现对 sliding window 的理解。
