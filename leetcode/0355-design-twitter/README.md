# 355. Design Twitter

## 我的思考
刷了约2小时。

核心思路其实很快就想清楚了：
- 关注关系：每个用户对应一个 `Set`，即 `Map<Integer, Set<Integer>>`
- 推文：使用链表按时间顺序记录
- 获取最近10条：使用堆模拟败者树（k路归并）

思维岔路：
很自然想到每个用户其实只需要保留最近10条推文，因此老推文可以直接删除。

但实现删除时遇到问题：
- 不能直接用 `Queue`，因为获取新闻流时需要遍历推文链表
- 也没有选择 `ArrayList`，因为删除复杂度较高（实际上长度最多10完全可以接受，但当时没有想到）

最后选择自定义链表结构，并在插入时维护长度。

## 卡点
写完主体逻辑后处理了一些空指针问题：
- 用户可能没有关注任何人
- 用户可能没有发过推文

这类边界情况其实应该在设计阶段提前考虑。
