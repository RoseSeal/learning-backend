# 2402. Meeting Rooms III

## 我的思考
单堆可以找到最快结束且房间号最小的房间，
但若下个会议开始较晚，最快结束的房间不一定是序号最小的。
→ 优化：while 循环把所有满足要求的房间释放再重新入堆
→ 问题：反复出入堆，效率和语义都不好
→ 最终：释放的房间不必马上入原堆，新建一个空闲房间堆
→ 双堆：freeRoom（按编号）+ meetingRoom（按结束时间）

## 关键点
- 资源调度问题 → 状态分离（空闲 vs 占用）→ 双堆
- 单堆能 AC 但语义不清晰，双堆才是正确建模

## 复杂度
- 时间：O(m log m + m log n)
- 空间：O(n)

## 卡点
- endTime会超出int范围，得用long
- 双堆解法中，meetingRoom堆仍然要保留终止时间相同则由房间序号决定顺序的逻辑，否则无空闲房间时从meetingRoom取出一个非空闲房间过程会错误
